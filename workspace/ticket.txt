TICKET
======
Summary: Two functions, `get_average` and `find_max`, contain logic errors leading to incorrect calculations.

Bugs found:
1. Function: get_average
   Problem: The function divides the sum of numbers by a hardcoded value of 10 instead of the actual number of elements in the input list.
   Expected: The function should divide the sum of `numbers` by `len(numbers)` to correctly calculate the average.

2. Function: find_max
   Problem: The `max_val` variable is initialized to an extremely high number (99999), preventing it from being updated by typical input values and causing the function to incorrectly return the initial high value.
   Expected: `max_val` should be initialized to the first element of the `numbers` list (assuming a non-empty list) or to a sufficiently low value (e.g., negative infinity) to ensure it is correctly updated by the first actual number.

End of ticket.
