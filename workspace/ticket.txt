TICKET
======
Summary: Several functions in the script have logic errors leading to incorrect output or ignored parameters.

Bugs found:
1. Function: flatten
   Problem: The `depth` parameter is provided but not utilized in the function's logic. The function always performs a full flatten, regardless of the specified `depth`.
   Expected: The `depth` parameter should control how many levels of nesting are flattened. For `depth=1`, only the top-level lists should be flattened, leaving nested lists one level deep.

2. Function: running_total
   Problem: The calculation of the running total incorrectly skips the second element (`transactions[1]`) of the input list. The loop starts from `transactions[2:]`.
   Expected: The running total should sum all elements in the `transactions` list, including `transactions[1]`.

3. Function: chunk_list
   Problem: The loop condition `len(lst) - size` prevents the last chunk (or chunks, if `len(lst)` is not perfectly divisible by `size`) from being processed and added to the result.
   Expected: The function should include all elements of the list, forming complete chunks and a final partial chunk if necessary. The loop should iterate up to `len(lst)`.

End of ticket.
