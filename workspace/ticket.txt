TICKET
======
Summary: Several list manipulation utility functions contain logic errors leading to incorrect output.

Bugs found:
1. Function: flatten
   Problem: The `depth` parameter is ignored, causing the function to always fully flatten the nested list regardless of the specified depth.
   Expected: The function should flatten the list only up to the specified `depth` level.

2. Function: running_total
   Problem: The function incorrectly calculates the sum by skipping the second element of the `transactions` list due to an incorrect slicing in the loop (`transactions[2:]`).
   Expected: The function should sum all elements in the `transactions` list.

3. Function: chunk_list
   Problem: The function fails to include the last chunk(s) of the list because the `range` upper bound (`len(lst) - size`) is too restrictive, causing the loop to terminate prematurely.
   Expected: The function should create all possible chunks of the specified `size`, including a potentially smaller final chunk if the list length is not perfectly divisible by `size`.

End of ticket.
