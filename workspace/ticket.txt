TICKET
======
Summary: The Dev agent is not given the latest patched code from previous attempts, leading to repeated fixes on the original script.

Bugs found:
1. Function: main
   Problem: In the Dev + QA loop, the `run_dev` function always receives the initial `code` (from `ORIGINAL_SCRIPT`) as its base. When QA finds issues and the ticket is updated with feedback, the Dev agent for subsequent attempts still works on the original script, not the code that was previously patched and reviewed. This prevents iterative refinement of the patch.
   Expected: After each successful `run_dev` call and writing to `PATCHED_SCRIPT`, the `code` variable should be updated with the content of the `patch` (i.e., the current state of the patched script). This updated `code` should then be passed to `run_dev` in the next iteration if QA fails.

End of ticket.
